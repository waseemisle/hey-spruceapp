Root Cause — Why You're Getting This Error

All API routes in Next.js run server-side on Vercel. Most routes initialize Firebase using the client SDK (firebase/firestore, firebase/app) — the same SDK used in the browser.

The problem: on the server, no user is logged in. The Firebase client SDK sends requests to Firestore with no auth token. Your Firestore rules require isSignedIn() — meaning request.auth != null — for every single collection. Without auth, every Firestore call fails with "Missing or insufficient permissions."

In the browser, users are logged in so this works fine. The bug only shows up in server-side API routes that initialize their own Firebase instance without signing in first.
